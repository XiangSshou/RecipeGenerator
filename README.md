# 营养搭配生成器 (Recipe Generator)

根据目标蛋白质、碳水、脂肪（克），从自定义食物列表中随机生成 3～5 种食物的搭配方案，并计算每种食物的用量（生重），使总营养接近目标。

纯前端单页应用，无需后端，打开即用。

## 功能

- **每日热量消耗**：按性别、年龄、体重、身高、活动量估算 BMR 与 TDEE（Mifflin-St Jeor + 活动系数）
- **目标营养素**：输入目标蛋白质、碳水、脂肪（克），自动显示预估总热量
- **食物来源**：上传 `.txt` 食物列表（格式见下）；上传后仅使用该列表，不再使用内置默认列表；支持下载示例文件
- **五个搭配槽位**：每个槽位可选择「随机」或按标签筛选，并可指定该槽位食物的最小/最大重量（克），为空则不限制；槽位 4、5 为「随机」时可能不参与当次搭配
- **生成搭配**：每次尝试随机选取食物并求解用量，最多 120 次，取满足各槽位重量限制且与目标 P/C/F 偏差最小的结果；展示时按结果重新计算营养与热量

## 使用方式

1. 用浏览器打开 `index.html`，或使用任意静态服务器（如 `npx serve .`）访问。
2. （可选）在「每日热量消耗」中填写身体数据，查看 BMR/TDEE。
3. 在「目标营养素」中填写目标蛋白质、碳水、脂肪（克）。
4. （可选）在「食物来源」中上传自己的 `.txt` 食物列表，或使用内置默认列表。
5. 在「五个搭配槽位」中为每个槽位选择来源（随机或标签）、具体食物（可选），以及最小/最大重量（可选）。
6. 点击「生成搭配」，查看推荐食物与用量。

## 食物列表格式（.txt）

每行一种食物，用英文竖线 `|` 分隔，格式：

```
名称 | 蛋白质(g) | 碳水(g) | 脂肪(g) | 标签1,标签2
```

- 每 100g 生重的营养数据；标签可选，多个标签用逗号分隔。
- 以 `#` 开头的行和空行会被忽略。
- 可在页面点击「下载示例文件」获取示例。

示例：

```
# 食物列表（每100g 生重）
大米 | 7.4 | 77 | 0.8 | 谷物,主食
鸡胸肉 | 31 | 0 | 3.6 | 蛋白质,肉类
```

## 技术说明

- 单文件 HTML + CSS + JavaScript，无构建步骤。
- 搭配求解：根据槽位得到 3～5 种食物后，用线性方程组（3×3 的 Cramer 法则）求解用量；若为 4 或 5 种食物，则固定部分用量再解前 3 种，并校验各槽位最小/最大重量限制。
